They feel difficult because they essentially do the same thing that we have learned so far this semester, but you have to do it in a completely different way. With practice, this will likely become second nature. Plus, React seems like it is better in many cirumstances.

"One of the most important aspects of React is the fact that you can create components, which are like custome, reusable HTML elements, to quickly and efficiently build user interfaces" (taniarascia).

From my readings on the differences between React and standard JavaScript, React creates the HTML while JS defines behavior of already created HTML files. React is also easier to update because it is in control of the HTML. JavaScript needs to go in to remove and add elements. React is also easier to maintain than the standard way we have approached web design thus far. The expansiveness of it is faster and easier.

The code below is from w3schools on react forms and has an example of it running here. This code prompts the user to input their name and displays it on screen when they do so. The hHTML file is basically empty and the building of it comes from the JS file with React. It creates the 'username' object separate the HTML, making this code very easy to change around as needed.

Code:
// index.js import React from 'react'; import ReactDOM from 'react-dom'; class MyForm extends React.Component { constructor(props) { super(props); this.state = { username: '' }; } myChangeHandler = (event) => { this.setState({username: event.target.value}); } render() { return ( <form> <h1>Hello {this.state.username}</h1> <p>Enter your name:</p> <input type='text' onChange={this.myChangeHandler} /> </form> ); } } ReactDOM.render(<MyForm />, document.getElementById('root')); //index.html <!DOCTYPE html> <html lang="en"> <head> <meta charset="utf-8" /> <meta name="viewport" content="width=device-width, initial-scale=1" /> <title>React App</title> </head> <body> <div id="root"></div> </body> </html> Return to Table of Contents...
